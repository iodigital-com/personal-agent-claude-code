# Archives

The graveyard — but a useful one. Completed projects, old journal years, retired areas, and superseded resources all live here.

---

## Structure

```
Archives/
├── Projects/   ← completed or abandoned projects
├── Areas/      ← retired areas of responsibility
├── Resources/  ← superseded or outdated reference notes
└── Journal/    ← old journal years
```

---

## Rules

- **Never delete** — archive instead
- When archiving, update the note's `status: archived` and add `archived: YYYY-MM-DD` to frontmatter
- Keep the original folder path structure (a project moves from `Projects/` to `Archives/Projects/`)
- Archived notes are low-priority for LLM retrieval — they exist for human search, not active context
